# Заявки с сайта → Telegram: настройка и обслуживание

Форма «Контакты» на сайте отправляет каждую заявку (имя, телефон, направление, сообщение) в Telegram-группу через бота. Здесь — как всё настроить с нуля и что делать, если уведомления перестали приходить.

## 1. Создать бота (2 минуты)

1. Открой в Telegram бота **@BotFather**.
2. Отправь команду `/newbot`.
3. Введи отображаемое имя (например, «Заявки с сайта»).
4. Введи юзернейм — должен заканчиваться на `bot` (например, `yuriki_leads_bot`).
5. BotFather пришлёт **токен** вида `123456789:AAAbbbCCC...` — сохрани его.

⚠️ **Токен = пароль.** Никому не показывать, не публиковать, не коммитить в git. Если утёк — в BotFather: `/mybots` → выбрать бота → API Token → Revoke.

## 2. Добавить бота в группу

1. Создай группу (или используй существующую), добавь туда всех, кто должен видеть заявки.
2. Добавь бота в группу как обычного участника (поиск по его юзернейму).

## 3. Узнать chat_id группы

1. Напиши в группу любое сообщение (например, «тест»).
2. Открой в браузере (подставь свой токен):
   ```
   https://api.telegram.org/bot<ТОКЕН>/getUpdates
   ```
3. В ответе найди `"chat":{"id":-100...}` — это и есть **chat_id** группы (отрицательное число, у супергрупп начинается с `-100`).

Альтернатива: добавь в группу бота **@getidsbot** — он сразу напишет id, потом его можно удалить.

Если нужно слать в несколько групп — перечисли id через запятую: `-100111,-100222`.

## 4. Прописать переменные

### На проде (Vercel)

1. [vercel.com](https://vercel.com) → проект **project-yuriki** → **Settings → Environment Variables**.
2. Добавь две переменные (окружение — Production, можно и Preview):
   - `TELEGRAM_BOT_TOKEN` — токен из шага 1
   - `TELEGRAM_CHAT_IDS` — chat_id из шага 3
3. Переменные применяются только после нового деплоя — сделай пуш в `main` (или Redeploy в дашборде Vercel).

### Локально (для разработки)

Скопируй `.env.example` в `.env.local` в корне проекта и впиши реальные значения. Файл `.env.local` в git не попадает.

## 5. Проверить

Отправь заявку через форму на сайте — в группу должно прийти сообщение:

> 🔔 **Новая заявка с сайта**
>
> **Имя:** …
> **Телефон:** +7…
> **Направление:** …
> **Сообщение:** …
>
> _дата, время (МСК)_

## Если уведомления перестали приходить

**Главное: заявки не теряются.** Каждая заявка дублируется в логи Vercel: проект → **Logs** (или Deployments → последний деплой → Logs), искать по слову `[contact]`.

Типовые причины и лечение:

| Симптом в логах | Причина | Что делать |
|---|---|---|
| `bot was kicked` / `chat not found` | Бота удалили из группы или chat_id неверный | Вернуть бота в группу; проверить chat_id через getUpdates |
| `group chat was upgraded to a supergroup` | Группа мигрировала, старый chat_id умер | Получить новый id (шаг 3, будет `-100...`) и обновить `TELEGRAM_CHAT_IDS` в Vercel + redeploy |
| `Unauthorized` (401) | Токен отозван или неверный | Взять актуальный токен в BotFather (`/mybots` → API Token), обновить в Vercel + redeploy |
| `не заданы TELEGRAM_BOT_TOKEN / TELEGRAM_CHAT_IDS` | Переменные не прописаны (например, после пересоздания проекта) | Шаг 4 |
| Ничего в логах нет | Заявка не дошла до сервера | Проверить, открывается ли сайт; посмотреть консоль браузера |

## Как это устроено (для разработчика)

- API-эндпоинт: `src/app/api/contact/route.ts` — валидация, антиспам (honeypot + rate-limit 5 заявок / 10 мин с IP), отправка через Telegram Bot API `sendMessage` (HTML-разметка, 1 повторная попытка при сбое).
- Форма: `src/components/sections/ContactForm.tsx` — клиентская валидация, чекбокс согласия на обработку ПД, состояния «Отправка…» / ошибка.
- Rate-limit хранится в памяти serverless-функции — защита best-effort. Если начнётся реальный спам — добавить Cloudflare Turnstile или Upstash Redis.
